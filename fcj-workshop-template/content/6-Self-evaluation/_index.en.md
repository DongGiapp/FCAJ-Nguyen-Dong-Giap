# Self-Assessment

During the internship and report preparation process, I had the opportunity to organize my knowledge of AWS, serverless architecture, AI/LLM, and technical documentation. The main work I completed includes the **Serverless Todo API** project proposal, a workshop using **Amazon API Gateway**, **AWS Lambda**, and **Amazon DynamoDB**, three AWS-related blog summaries, and two FCAJ Community Day event reports.

## Completed Work

- Completed the **Project Proposal** for the Serverless Todo API project.
- Built a 6-part workshop: overview, prerequisites, DynamoDB table creation, Lambda functions, API Gateway testing, and resource cleanup.
- Wrote three blog sections about Amazon Bedrock Intelligent Prompt Routing, AWS Lambda serverless backend, and the benefits of Serverless.
- Documented two FCAJ Community Day events, including agenda, key topics, lessons learned, and personal reflections.
- Updated the Hugo report site with bilingual Vietnamese/English core content, ensuring that the overview, architecture, and workshop steps are translated accurately and clearly.

## Knowledge and Skills Gained

Through this process, I gained a better understanding of serverless backend architecture on AWS. I learned the role of API Gateway for REST endpoints, Lambda for business logic, and DynamoDB for NoSQL data storage.

I also learned more about modern AI and cloud topics such as LLMs, prompt routing, multi-agent systems, CloudFront, MCP, AI assistants, and event-driven architecture. Writing blogs and event recaps helped me improve my ability to read technical material, summarize key ideas, and present information clearly.

## Challenges

The biggest challenge was organizing many different topics into one consistent report. Concepts such as IAM permissions, API Gateway integration, Lambda execution flow, DynamoDB schema design, and serverless cost models required time to understand and connect with the practical workshop.

Working with Hugo also required attention to folder structure, front matter, image paths, and menu order. Through the editing process, I learned how to manage technical content more carefully and think more about the reader experience.
